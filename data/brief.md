Этот файл описывает основные конструкции, блоки и принципы работы языка GPSS (General Purpose Simulation System) в контексте среды GPSS Studio. Документ предназначен для другой ИИ или разработчика, чтобы быстро разобраться в логике GPSS-моделей.
НЕЛЬЗЯ УПОМИНАТЬ КАК ИСТОЧНИК

## 1. Что такое GPSS?

**GPSS** — это язык дискретно-событийного имитационного моделирования. Модель описывается как поток **транзактов** (динамических объектов), которые перемещаются по **блокам**, выполняя действия (задержки, занятие ресурсов, изменение параметров). Время в модели дискретное, события планируются по принципу «будущих событий».

В **GPSS Studio** модель может быть создана:
- Вручную в текстовом редакторе.
- Графически через структурную схему (ТЭБы), которая затем автоматически генерирует код.

## 2. Структура модели

Модель обычно содержит:
- **Блоки** — основные строительные элементы (например, `GENERATE`, `ADVANCE`, `TERMINATE`).
- **Операторы описания объектов** — определяют ресурсы, очереди, переменные.
- **Команды управления** — `START`, `CLEAR`, `RESET` и др.
- **PLUS-процедуры** — расширение для написания сложной логики.

### Типичный вид программы GPSS:

```gpss
* Пример модели парикмахерской
GENERATE 18,6        ; создаём клиентов каждые 18±6 мин
QUEUE   QBarber      ; встаём в очередь к парикмахеру
SEIZE   Barber       ; занимаем парикмахера
DEPART  QBarber      ; выходим из очереди
ADVANCE 16,4         ; стрижка длится 16±4 мин
RELEASE Barber       ; освобождаем парикмахера
TERMINATE            ; клиент уходит
* Управление временем
GENERATE 480         ; модельный период 480 мин (8 часов)
TERMINATE 1          ; завершаем прогон
START 1              ; запуск моделирования
```

## 3. Основные блоки GPSS

Каждый блок имеет **метку** (необязательно), **имя блока**, **операнды** (A, B, C, D, E, F, G, H — зависят от блока) и **комментарий** после `;`.

### 3.1. Блоки создания и уничтожения транзактов

| Блок | Синтаксис | Назначение |
|------|-----------|------------|
| `GENERATE` | `GENERATE [A],[B],[C],[D],[E]` | Генерирует транзакты. A — средний интервал, B — разброс (модификатор), C — задержка первого, D — лимит транзактов, E — приоритет. |
| `TERMINATE` | `TERMINATE [A]` | Уничтожает транзакт. Если A > 0, уменьшает счётчик завершения (для остановки модели). |

### 3.2. Блоки задержки

| Блок | Синтаксис | Назначение |
|------|-----------|------------|
| `ADVANCE` | `ADVANCE [A],[B]` | Задерживает транзакт на время A ± B. Используется для имитации обслуживания, движения и т.п. |

### 3.3. Блоки очередей и ресурсов

| Блок | Синтаксис | Назначение |
|------|-----------|------------|
| `QUEUE` | `QUEUE [A]` | Помещает транзакт в очередь с именем A (накапливает статистику). |
| `DEPART` | `DEPART [A]` | Удаляет транзакт из очереди A. |
| `SEIZE` | `SEIZE [A]` | Занимает устройство A (одноканальный ресурс). Если занято, транзакт ждёт. |
| `RELEASE` | `RELEASE [A]` | Освобождает устройство A. |
| `ENTER` | `ENTER [A]` | Занимает память A (многоканальный ресурс). |
| `LEAVE` | `LEAVE [A]` | Освобождает память A. |

### 3.4. Блоки управления потоком

| Блок | Синтаксис | Назначение |
|------|-----------|------------|
| `TRANSFER` | `TRANSFER [Mode],B,C` | Передаёт транзакт на метку B или C в зависимости от режима (безусловный, статистический, условный). |
| `GATE` | `GATE [Type] [A]` | Проверяет состояние ресурса A (например, свободен ли логический ключ, память, устройство). Если условие не выполнено, транзакт задерживается. |
| `TEST` | `TEST [Type] A,B` | Проверяет условие между операндами A и B (равно, больше и т.д.). Если ложно, транзакт может пойти по альтернативному пути (зависит от режима). |
| `LOOP` | `LOOP A,B` | Организует цикл: A — счётчик, B — метка начала цикла. |

### 3.5. Блоки для работы с данными

| Блок | Синтаксис | Назначение |
|------|-----------|------------|
| `SAVEVALUE` | `SAVEVALUE [A],B` | Сохраняет значение B в ячейку A (сохраняемая величина). |
| `ASSIGN` | `ASSIGN [A],B` | Присваивает параметру транзакта A значение B (или выражение). |
| `INITIAL` | `INITIAL [type] [name],value` | Задаёт начальное значение объекту (ячейке, памяти, матрице). |

## 4. Операторы описания объектов

Эти операторы размещаются вне блоков (обычно в начале модели) и определяют используемые ресурсы.

```gpss
* Устройство
Barber   EQU 1

* Память (многоканальное устройство)
Parking  STORAGE 10

* Очередь (не требует описания, просто используется в блоках QUEUE/DEPART)

* Ячейка (сохраняемая величина)
X        EQU 1
INITIAL  X$X, 0

* Переменная (вычисляемое выражение)
Y        VARIABLE  (AC1 + 5)

* Логический ключ (переключатель)
LOGIC    L1
INITIAL  LOGIC L1, 0   ; выключен
```

## 5. Системные числовые атрибуты (СЧА)

СЧА — это предопределённые значения, которые можно использовать в выражениях (например, в операндах блоков).

| СЧА | Значение |
|-----|----------|
| `AC1` | Модельное время в текущих единицах |
| `XN1` | Номер текущего транзакта |
| `X$X` | Значение ячейки с именем X |
| `Q$Q` | Текущая длина очереди Q |
| `S$U` | Занятость устройства U (0/1) |
| `F$F` | Состояние логического ключа F (0/1) |

## 6. Команды управления моделированием

В GPSS Studio команды вводятся в окне **«Журнал моделирования»** после трансляции.

| Команда | Действие |
|---------|----------|
| `START A` | Запускает моделирование до завершения (A — счётчик завершения). |
| `STEP A` | Выполняет A шагов (входов в блоки). |
| `HALT` | Останавливает моделирование. |
| `CLEAR` | Сбрасывает все статистики и удаляет транзакты. |
| `RESET` | Сбрасывает статистики без удаления транзактов. |
| `SHOW expr` | Выводит значение выражения. |

## 7. PLUS — язык расширения

PLUS позволяет писать процедуры и выражения, которые дополняют GPSS. Используется для сложной логики, вычислений и управления экспериментом.

Пример:

```plus
PROCEDURE MyProc(Arg1, Arg2)
  IF Arg1 > Arg2
    RETURN Arg1
  ELSE
    RETURN Arg2
  END
END
```

PLUS-процедуры можно вызывать в операндах блоков, например:

```gpss
SAVEVALUE MaxVal, (MyProc(5, 10))
```

## 8. Пример полной модели (парикмахерская)

```gpss
* Парикмахерская (один мастер, очередь)
GENERATE 18,6        ; интервал прихода 18±6 мин
QUEUE   QBarber
SEIZE   Barber
DEPART  QBarber
ADVANCE 16,4         ; время обслуживания 16±4 мин
RELEASE Barber
TERMINATE
* Управление временем (8 часов = 480 мин)
GENERATE 480
TERMINATE 1
START 1
```

После прогона GPSS формирует **стандартный отчёт** со статистикой по очередям и устройствам.

## 9. Особенности GPSS Studio

- **Структурные схемы** — модель собирается из типовых элементов (ТЭБ), которые затем транслируются в GPSS-код.
- **Тестирование ТЭБов** — позволяет проверять отдельные блоки схемы.
- **Интерактивный отладчик** — пошаговое выполнение с визуализацией.
- **Интеграция с C#** — вызов внешних методов.

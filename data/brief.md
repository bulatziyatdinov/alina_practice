## 2. Базовая структура программы

Программа обычно состоит из:
- Описаний объектов (устройства, очереди, памяти, ячейки, переменные)
- Блоков, формирующих логику модели
- Блоков управления прогоном (GENERATE/TERMINATE для времени моделирования)

Пример простейшей модели:

```
* Описание устройств (ресурсов)
Кассир  DEVICE

* Генерация транзактов
GENERATE  10,2            ; создавать транзакт каждые 10±2 минуты

* Занятие ресурса
QUEUE     ОчередьККассиру ; войти в очередь
SEIZE     Кассир          ; захватить устройство
DEPART    ОчередьККассиру ; выйти из очереди

* Обслуживание
ADVANCE   16,4            ; задержка 16±4 минуты

* Освобождение ресурса и завершение
RELEASE   Кассир
TERMINATE                  ; уничтожить транзакт

* Управление временем прогона
GENERATE  480              ; один транзакт в момент 480 (8 часов)
TERMINATE 1                ; завершить прогон

START 1                    ; запустить моделирование
```

## 3. Базовые понятия

- **Транзакт** — динамический объект, движущийся по модели. Может иметь параметры.
- **Блок** — элемент модели, выполняющий действие (задержка, занятие ресурса и т.д.).
- **Устройство** — одноканальный ресурс (занят/свободен).
- **Память** — многоканальный ресурс (занимает несколько единиц ёмкости).
- **Очередь** — статистический объект для сбора данных о времени ожидания.
- **Ячейка** — числовая переменная для хранения значений.
- **СЧА** — системный числовой атрибут (например, Q$Очередь — длина очереди).

## 4. Синтаксис блоков

Общий формат:
`[метка] БЛОК ПараметрА, ПараметрB, ... ; комментарий`

- Метка необязательна, но полезна для переходов
- Параметры разделяются запятыми
- Комментарий после `;`
- Пустые параметры означают значение по умолчанию

## 5. Основные блоки

### GENERATE — создание транзактов
`GENERATE [Среднее], [Модификатор], [Задержка], [Макс_транзактов]`
- Создаёт транзакты с интервалом Среднее ± Модификатор
- Пример: `GENERATE 18,6` — интервал 18±6

### TERMINATE — уничтожение транзакта
`TERMINATE [Счетчик]`
- Если Счетчик > 0, уменьшает счётчик завершения START
- Пример: `TERMINATE 1`

### QUEUE / DEPART — учёт времени в очереди
`QUEUE Имя_очереди`
`DEPART Имя_очереди`
- Пара между QUEUE и DEPART фиксирует время ожидания

### SEIZE / RELEASE — занятие/освобождение устройства
`SEIZE Имя_устройства`
`RELEASE Имя_устройства`
- Устройство — одноканальный ресурс

### ENTER / LEAVE — занятие/освобождение памяти
`ENTER Имя_памяти [, Количество]`
`LEAVE Имя_памяти [, Количество]`
- Память — многоканальный ресурс с ёмкостью STORAGE

### ADVANCE — временная задержка
`ADVANCE [Среднее], [Модификатор]`
- Задерживает транзакт на время Среднее ± Модификатор

### TRANSFER — передача управления
`TRANSFER [Режим], [Метка], [Метка2], [Вероятность]`
- Безусловный: `TRANSFER ,Метка`
- Статистический: `TRANSFER 0.5,,Метка2` (50% на Метка2)
- По условию: `TRANSFER BOTH,Метка1,Метка2`

### ASSIGN — присвоить параметр транзакту
`ASSIGN [Номер_параметра], [Значение]`
- Параметры транзакта — это его локальные переменные
- Пример: `ASSIGN 1,5` — параметру 1 присвоить 5

### SAVEVALUE — запись в ячейку
`SAVEVALUE Имя_ячейки, [Значение]`
- Увеличить: `SAVEVALUE Счетчик+,1`
- Присвоить: `SAVEVALUE Счетчик,10`

### TEST — проверка условия
`TEST [Условие], [ОперандA], [ОперандB], [Метка]`
- Условия: E (равно), NE (не равно), L (меньше), LE (меньше или равно), G (больше), GE (больше или равно)
- При ложном условии переход на Метка

### GATE — проверка состояния объекта
`GATE [Режим], Имя_объекта [, Метка]`
- Режимы: LS (память свободна), NU (устройство свободно), SN (ячейка не равна 0)
- Пример: `GATE NU Кассир` — если кассир свободен, идти дальше

### LOOP — цикл
`LOOP [Параметр], [Метка]`
- Уменьшает параметр на 1 и переходит на Метку, пока параметр > 0

### MATRIX — описание матрицы
`MATRIX Имя_матрицы, [Строки], [Столбцы]`
- Обращение: `MX$Имя(строка,столбец)`

### INITIAL — начальное значение
`INITIAL [Объект], [Значение]`
- Пример: `INITIAL MX$Матрица(1,1),5`

### STORAGE — ёмкость памяти
`Имя_памяти STORAGE [Ёмкость]`

### VARIABLE — переменная
`Имя VARIABLE [Выражение]`
- Пример: `X VAR 10` — но в GPSS используется синтаксис:
  `Имя VARIABLE (AC1+5)`

### FUNCTION — функция пользователя
`Имя FUNCTION Аргумент, Тип`
- Типы: D1 (дискретная), C1 (непрерывная)
- Список пар: точка1/значение1, точка2/значение2, ...

## 6. Типовые объекты

**Устройство** (SEIZE/RELEASE)
- Объявляется просто по имени: `Кассир DEVICE`

**Память** (ENTER/LEAVE)
- Объявляется: `Залы STORAGE 100` (ёмкость 100)

**Очередь** (QUEUE/DEPART)
- Объявляется по имени: `ОчередьККассиру QUEUE`

**Ячейка** (SAVEVALUE)
- Объявляется по имени: `Счетчик EQU 0`

**Именованная величина** (VARIABLE)
- `Время VARIABLE (AC1*2)`

## 7. Пример полной структуры модели

```
* ===== Описания объектов =====
Кассир DEVICE
Очередь QUEUE
Вместимость STORAGE 10

* ===== Генерация клиентов =====
GENERATE 18,6,,100      ; 100 клиентов

* ===== Прибытие и очередь =====
QUEUE Очередь
SEIZE Кассир
DEPART Очередь

* ===== Обслуживание =====
ADVANCE 16,4

* ===== Освобождение =====
RELEASE Кассир
TERMINATE

* ===== Управление временем =====
GENERATE 480
TERMINATE 1

* ===== Запуск =====
START 1
```

## 8. Параметры блоков (краткая справка)

| Блок | Параметры |
|------|-----------|
| GENERATE | A=среднее, B=модификатор, C=первая задержка, D=макс. транзактов |
| ADVANCE | A=среднее, B=модификатор |
| QUEUE | A=имя очереди, B=приоритет |
| SEIZE | A=имя устройства |
| RELEASE | A=имя устройства |
| ENTER | A=имя памяти, B=количество |
| LEAVE | A=имя памяти, B=количество |
| TRANSFER | A=режим, B=метка, C=метка2, D=вероятность |
| TEST | A=условие, B=операнд1, C=операнд2, D=метка |
| ASSIGN | A=номер параметра, B=значение |
| SAVEVALUE | A=имя ячейки, B=значение, C=режим |
| LOOP | A=параметр, B=метка |
| GATE | A=режим, B=имя, C=метка |
| TERMINATE | A=счетчик |

## 9. Команды управления моделированием

- `START [Счетчик]` — запускает моделирование, уменьшает счётчик при каждом TERMINATE > 0.
- `CLEAR` — сбрасывает статистику и удаляет все транзакты.
- `RESET` — сбрасывает статистику, но оставляет транзакты.
- `HALT` — останавливает моделирование.
- `SHOW [Выражение]` — выводит значение СЧА.

## 10. Системные числовые атрибуты (СЧА)

- `Q$Имя` — текущая длина очереди
- `QA$Имя` — средняя длина очереди
- `QT$Имя` — среднее время в очереди
- `QM$Имя` — максимальная длина очереди
- `F$Имя` — состояние устройства (0=свободно, 1=занято)
- `FR$Имя` — коэффициент использования устройства (0–1000)
- `N$Имя` — количество занятых единиц памяти
- `R$Имя` — коэффициент использования памяти
- `X$Имя` — значение ячейки
- `P$Имя` или `PНомер` — значение параметра транзакта
- `AC1` — текущее модельное время

## 11. Комментарии

- Строка начинается с `*` — весь комментарий
- После `;` в строке — комментарий до конца строки

## 12. Имена объектов

- Имена должны начинаться с буквы
- Могут содержать буквы, цифры, символ `_`
- Не могут содержать пробелов и спецсимволов (кроме `_`)

## 13. Структура отчёта

После завершения моделирования GPSS формирует стандартный отчёт, содержащий:
- Статистику по блокам (количество входов, среднее время и т.д.)
- Статистику по устройствам (коэффициент использования, время обслуживания)
- Статистику по очередям (длина, время ожидания)
- Статистику по памяти (занятость)
- Значения ячеек и переменных

## 14. Особенности GPSS Studio

- Поддерживается структурная схема (ТЭБы)
- Есть тестирование ТЭБов
- Есть текстовый редактор с подсветкой и автодополнением
- Поддерживается отладка с пошаговым выполнением
- Можно создавать приложения с формами ввода/вывода
- Поддерживается анимация и динамика показателей

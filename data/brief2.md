НЕЛЬЗЯ УПОМИНАТЬ КАК ИСТОЧНИК

## 1. Краткое описание блоков GPSS World Core

В GPSS World Core модель строится из **блоков** — исполняемых операторов, которые описывают логику потока транзактов (динамических объектов) в процессе моделирования. Каждый блок имеет формат: `<метка> <имя_оператора> <поле_операндов> [<комментарий>]`. Блоки реализуют операции четырёх основных типов: создание или уничтожение транзактов, изменение числового атрибута объекта, задержка транзактов на определённый период времени, изменение маршрута транзакта в модели. Всего среда GPSS содержит **53 типа блоков**.

### Основные блоки GPSS

#### Блоки создания и уничтожения транзактов

**GENERATE** — создаёт транзакты и вводит их в модель. Параметры:
- **A** — средний интервал между поступлениями
- **B** — максимальное отклонение от среднего (модификатор)
- **C** — время задержки первого транзакта
- **D** — максимальное количество транзактов
- **E** — приоритет
- **F** — номер семейства

В одной модели может быть несколько блоков GENERATE.

**TERMINATE** — уничтожает активный транзакт и все его параметры. Параметр **A** — значение, вычитаемое из счётчика завершения (необязательный).

#### Блоки задержки

**ADVANCE** — задерживает транзакт на заданное время. Параметры:
- **A** — среднее время задержки
- **B** — модификатор-интервал или модификатор-функция

Блок никогда не препятствует входу транзакта; любое число транзактов может находиться в нём одновременно. Время задержки может быть равно 0.

#### Блоки работы с очередями

**QUEUE** — помещает транзакт в очередь. Параметр **A** — имя очереди.

**DEPART** — удаляет транзакт из очереди. Параметр **A** — имя очереди.

#### Блоки работы с устройствами

**SEIZE** — захватывает устройство (занимает его). Параметр **A** — имя устройства.

**RELEASE** — освобождает устройство. Параметр **A** — имя устройства.

#### Блоки передачи управления

**TRANSFER** — передаёт транзакт в другой блок. Параметры:
- **A** — режим перехода (BOTH, ALL, и др.)
- **B**, **C**, **D** — метки блоков назначения

#### Другие важные блоки

**SPLIT** — создаёт копии текущего транзакта.

**ASSEMBLE** — собирает несколько транзактов одного семейства в один.

**ENTER / LEAVE** — вход в многоканальное устройство (память) и выход из него.

**GATE** — проверка состояния оборудования или блоков.

**LINK / UNLINK** — работа со списками пользователя.

**SAVEVALUE** — сохранение значения в ячейке.

**MATRIX** — описание матрицы.

**FUNCTION** — описание функции пользователя.

**TABLE** — описание таблицы-гистограммы.

**INITIAL** — задание начальных значений.

**STORAGE** — описание ёмкости многоканального устройства.

## Полный справочник блоков языка GPSS

Язык GPSS (General Purpose Simulation System) содержит **53 типа блоков**, каждый из которых выполняет строго определённую функцию в имитационной модели. Блоки являются основными строительными элементами GPSS-программы — это операторы, отображающие события, происходящие в моделируемой системе при перемещениях транзактов.

**Общий формат записи блока:**
```
<метка> <имя_блока> <поле_операндов> [<комментарий>]
```

Метка может занимать позиции, начиная со второй, имя оператора — с восьмой, поле операндов — с девятнадцатой. Операнды отделяются друг от друга запятыми. Если какой-то операнд отсутствует, соответствующая ему запятая остаётся. Комментарий отделяется от операндов пробелом.

## 1. Блоки моделирования и управления программой

### SIMULATE
**Назначение:** Блок моделирования. Должен быть первым блоком программы. Если он отсутствует, выполнение модели не производится.

### END
**Назначение:** Блок конца программы. Ставится последним в модели.

### START
**Назначение:** Блок начала моделирования. Запускает процесс имитации.
- **A** — начальное значение счётчика завершений
- **B** — признак подавления печати (NP — стандартная печать не производится)
- **C** — число завершений, через которое будет выдаваться промежуточная печать
- **D** — признак печати цепей (1 — печать производится)

## 2. Блоки генерации и уничтожения транзактов

### GENERATE
**Назначение:** Создаёт транзакты и вводит их в модель. Является источником потока транзактов. В одной модели может быть несколько блоков GENERATE, работающих параллельно.

**Синтаксис:** `GENERATE A, B, C, D, E, F, G`

| Операнд | Назначение | Значение по умолчанию |
|---------|------------|----------------------|
| **A** | Средний интервал времени между последовательными поступлениями транзактов | 0 |
| **B** | Модификатор интервала (разброс относительно A). Может быть числом (равномерное распределение) или функцией | 0 |
| **C** | Момент времени появления первого транзакта | 1 |
| **D** | Максимальное количество транзактов, которое может быть сгенерировано | ∞ |
| **E** | Приоритет транзактов (от 0 до 127) | 0 |
| **F** | Номер семейства транзактов | — |
| **G** | Дополнительный параметр | — |

**Примеры:**
- `GENERATE 10,5` — интервалы между поступлениями равномерно распределены в диапазоне от 5 до 15
- `GENERATE 10` — постоянный интервал 10 единиц времени
- `GENERATE 25,15` — интервал от 10 до 40 единиц времени

**Особенности:** Последующее сообщение не генерируется до тех пор, пока предыдущее не покидает блок GENERATE. Содержимое счётчика текущего числа находящихся в блоке сообщений (Wj) никогда не превышает 1.

### TERMINATE
**Назначение:** Уничтожает транзакт, удаляя его из модели. В момент входа транзакта в этот блок освобождается память, выделенная под транзакт.

**Синтаксис:** `TERMINATE A`

| Операнд | Назначение | Значение по умолчанию |
|---------|------------|----------------------|
| **A** | Значение, на которое уменьшается счётчик завершений | 0 |

**Особенности:** Если поле A не определено или равно 0, транзакты, проходящие через такой блок, не уменьшают содержимого счётчика завершений. Блок TERMINATE всегда принимает транзакты.

## 3. Блоки задержки

### ADVANCE
**Назначение:** Задерживает продвижение активного транзакта на заданное время. Блок никогда не препятствует входу транзакта — любое число транзактов может находиться в нём одновременно.

**Синтаксис:** `ADVANCE A, B`

| Операнд | Назначение | Значение по умолчанию |
|---------|------------|----------------------|
| **A** | Среднее время задержки (обязательный параметр) | — |
| **B** | Модификатор, задающий разброс времени задержки | 0 |

**Типы модификаторов:**
1. **Модификатор-интервал** — если B является числом, то время задержки равномерно распределено в интервале [A-B, A+B]
2. **Модификатор-функция** — если B является функцией, то время задержки = A × значение функции

**Примеры:**
- `ADVANCE 10` — задержка ровно на 10 единиц времени
- `ADVANCE 15,5` — задержка равномерно распределена от 10 до 20 единиц
- `ADVANCE 10, FN$TOBR` — задержка = 10 × значение функции TOBR

## 4. Блоки работы с очередями

### QUEUE
**Назначение:** Помещает транзакт в очередь и фиксирует вход. При входе в блок увеличивается счётчик входов очереди и текущая длина очереди.

**Синтаксис:** `QUEUE A, B`

| Операнд | Назначение |
|---------|------------|
| **A** | Имя очереди (обязательный параметр) |
| **B** | Количество единиц, занимаемых в очереди (по умолчанию 1) |

### DEPART
**Назначение:** Обеспечивает выход транзакта из очереди. Уменьшает текущую длину очереди.

**Синтаксис:** `DEPART A, B`

| Операнд | Назначение |
|---------|------------|
| **A** | Имя очереди (обязательный параметр) |
| **B** | Количество освобождаемых единиц (по умолчанию 1) |

**Особенность:** Блоки QUEUE и DEPART используются как пара для сбора статистики об очередях. Статистика собирается в моменты входа и выхода транзактов.

## 5. Блоки работы с устройствами (приборами)

### SEIZE
**Назначение:** Занимает устройство (прибор). Если устройство уже занято другим транзактом, входящий транзакт останавливается в предыдущем блоке и ожидает освобождения.

**Синтаксис:** `SEIZE A`

| Операнд | Назначение |
|---------|------------|
| **A** | Имя устройства (обязательный параметр) |

### RELEASE
**Назначение:** Освобождает ранее занятое устройство.

**Синтаксис:** `RELEASE A`

| Операнд | Назначение |
|---------|------------|
| **A** | Имя устройства (обязательный параметр) |

### PREEMPT
**Назначение:** Захватывает прибор с возможностью прерывания текущего обслуживания.

**Синтаксис:** `PREEMPT A, B`

| Операнд | Назначение |
|---------|------------|
| **A** | Имя устройства |
| **B** | Режим прерывания |

### RETURN
**Назначение:** Возвращает захваченный прибор.

## 7. Блоки проверки условий

### GATE
**Назначение:** Проверяет состояние элемента (логического ключа, устройства, накопителя) и в зависимости от результата либо пропускает транзакт дальше, либо задерживает его.

**Синтаксис:** `GATE (X) A`

**Для логических ключей:**
- `GATE (LS) A` — проверка, что ключ A установлен (1)
- `GATE (LR) A` — проверка, что ключ A сброшен (0)

**Для устройств:**
- `GATE (U) A` — устройство A занято
- `GATE (NU) A` — устройство A свободно
- `GATE (I) A` — устройство A прервано
- `GATE (NI) A` — устройство A не прервано

**Для накопителей (памятей):**
- `GATE (SE) A` — накопитель A пуст
- `GATE (SF) A` — накопитель A заполнен
- `GATE (SNE) A` — накопитель A не пуст
- `GATE (SNF) A` — накопитель A не заполнен

### TEST
**Назначение:** Проверяет соотношение между двумя операндами и направляет транзакт в следующий блок при выполнении условия или по указанному адресу при невыполнении.

**Синтаксис:** `TEST (X) A, B, C`

| Операнд | Назначение |
|---------|------------|
| **X** | Оператор отношения: E (равно), NE (не равно), G (больше), GE (больше или равно), L (меньше), LE (меньше или равно) |
| **A** | Первый операнд |
| **B** | Второй операнд |
| **C** | Метка блока для перехода при невыполнении условия |

## 8. Блоки передачи управления

### TRANSFER
**Назначение:** Изменяет маршрут движения транзакта, передавая его в любой другой блок модели. Является основным средством организации ветвлений.

**Синтаксис:** `TRANSFER A, B, C, D`

| Режим | Описание |
|-------|----------|
| **Безусловный** | `TRANSFER ,B` — транзакт всегда переходит в блок B |
| **Статистический** | `TRANSFER P,B,C` — транзакт с вероятностью P переходит в блок B, иначе в C |
| **BOTH** | `TRANSFER BOTH,B,C` — транзакт сначала пытается войти в B, если невозможно — в C |
| **ALL** | `TRANSFER ALL,B` — транзакт пытается войти в первый доступный блок из списка |

**Примеры:**
- `TRANSFER ,LABEL1` — безусловный переход к метке LABEL1
- `TRANSFER 0.3,LABEL1,LABEL2` — с вероятностью 0.3 в LABEL1, иначе в LABEL2
- `TRANSFER BOTH,LABEL1,LABEL2` — сначала попытка войти в LABEL1

### LOOP
**Назначение:** Организует цикл — повторение группы блоков заданное число раз.

**Синтаксис:** `LOOP A, B`

| Операнд | Назначение |
|---------|------------|
| **A** | Счётчик повторений (номер параметра транзакта) |
| **B** | Метка начала цикла |

**Принцип работы:** При входе в LOOP значение параметра A уменьшается на 1. Если результат больше 0, транзакт переходит к блоку с меткой B. Если результат равен 0, транзакт переходит к следующему блоку.

## 10. Блоки работы с ансамблями транзактов

### SPLIT
**Назначение:** Создаёт копии (сплиты) входящего транзакта.

**Синтаксис:** `SPLIT A, B`

| Операнд | Назначение |
|---------|------------|
| **A** | Количество создаваемых копий |
| **B** | Метка блока, куда направляются копии |

**Особенность:** Основной транзакт переходит в следующий блок, а созданные копии направляются по указанному адресу.

### ASSEMBLE
**Назначение:** Собирает A транзактов одного ансамбля. В следующий блок пропускает первый (собранный) транзакт, остальные уничтожает.

**Синтаксис:** `ASSEMBLE A`

| Операнд | Назначение |
|---------|------------|
| **A** | Количество транзактов, которое необходимо собрать |

### GATHER
**Назначение:** Собирает A транзактов одного ансамбля и пропускает их одновременно в следующий блок.

**Синтаксис:** `GATHER A`

| Операнд | Назначение |
|---------|------------|
| **A** | Количество транзактов для сбора |

**Отличие от ASSEMBLE:** В GATHER все собранные транзакты продолжают движение одновременно, а не объединяются в один.

### MATCH
**Назначение:** Обеспечивает синхронизацию транзактов одного ансамбля.

## Сводка системных числовых атрибутов (СЧА)

| СЧА | Описание |
|-----|----------|
| **AC1** | Текущее модельное время |
| **RN1** | Случайная величина, равномерно распределённая в [0,1] |
| **FN** | Функция |
| **P** | Параметр транзакта |
| **Q** | Текущая длина очереди |
| **F** | Состояние устройства (0 — свободно, 1 — занято) |
| **S** | Объём занятой памяти в накопителе |
| **V** | Целочисленная переменная |
| **FV** | Вещественная переменная |
| **BV** | Булева переменная |
| **X** | Хранимая переменная |
| **K** | Константа |

**Формат записи СЧА:** `<СЧА>$<идентификатор>`
- `Q$ORD` — текущая длина очереди ORD
- `FN$COS` — значение функции COS
- `F$PRINTER` — состояние устройства PRINTER

## Задача 2

![Python](https://img.shields.io/badge/Python-3.12+-blue?logo=python&logoColor=white)
![Ollama](https://img.shields.io/badge/Ollama-0.30+-orange?logo=ollama&logoColor=white)
![ruff](https://img.shields.io/badge/ruff-linter-green?logo=ruff&logoColor=white)

**Цель:** разработать ИИ-консультанта на основе LLM, который отвечает на вопросы 
пользователей по работе с ALINA GPSS на основе руководства пользователя, 
а после завершения Задачи 1 – также на основе библиотеки эталонных примеров. 
Работа состоит из двух частей: 
- Часть 1 – локальное консольное приложение
- Часть 2 – чат-бот на внешней платформе

Файлы проекта по Задаче 2 лежат в репозитории в папке `alina_rag`.

В рамках работы были выполнены задачи:
- Разработан консультант на основе LLM и RAG по работе с ALINA GPSS (GPSS Studio)
- Разработан консольный интерфейс для общения с LLM
- Разработан telegram-бот для общения с LLM

## Требования

- Python 3.12+
- [Ollama](https://ollama.com/)
- Поддерживаются ОС Windows, Linux. macOS не проверялся, но по идее тоже работает
- Рекомендуемые системные требования: GPU с 8 ГБ VRAM (например, 3060 Ti), 
CPU 4 ядра/8 потоков, 8 ГБ ОЗУ. 

Знания системы будут извлекаться из файлов, лежащих в папке `data`.

_Ollama требуется для запуска локальных ИИ-моделей. От размера ИИ-модели зависит
качество работы консультанта. В рамках экспериментов было решено использовать
[FieldMouse-AI/qwen3.5:9b-instruct](https://ollama.com/FieldMouse-AI/qwen3.5:9b),
потому что качество её ответов на достаточно высоком уровне и у этой версии qwen3.5:9b
убраны мультимодальные возможности (обработка изображений, звуковых файлов и т.д.),
то есть работа ведётся лишь с текстом, что значительно уменьшает размер образа 
на диске, потребление VRAM и увеличивает скорость работы модели._

_Если ваш компьютер не справляется с работой данной модели, то всегда можно поставить
модель попроще, например, `qwen2.5:1.5b` в файле `.env`, но качество ответов упадёт._

## Установка и запуск

1. Скачайте модели

```bash
ollama pull FieldMouse-AI/qwen3.5:9b
ollama pull nomic-embed-text
```

2. Запустите Ollama

```bash
ollama serve
```

3. Скачайте проект 

```bash
git clone https://github.com/StPrep/Praktika2026.git
```

4. Перейдите в папку `Praktika2026/alina_rag`

```bash
cd Praktika2026/alina_rag
```

5. Создайте `.env` файл. Внесите в него свои настройки

```bash
cp example.env .env
```

6. Установите uv

```bash
pip install uv
```

7. Установите зависимости

```bash
uv sync --no-dev
```

Для разработки: 

```bash
uv sync
```

8. Запустите индексацию и дождитесь её конца.

```bash
uv run index.py
```

Займет некоторое время в зависимости от мощности железа.
Файлы для индексации находятся в папке `data`, это знания системы.

9. Запустите RAG (ИИ-консультанта) в консольном режиме

```bash
uv run main.py
```

10. Запуск бота для telegram (ОБЯЗАТЕЛЬНО УКАЗАТЬ КЛЮЧ БОТА в .env)

```bash
uv run bot.py
```

## Конфигурация (пример из `example.env`)

```env
# Модели и подключение
EMBEDDING_MODEL=nomic-embed-text
LLM_MODEL=FieldMouse-AI/qwen3.5:9b
OLLAMA_BASE_URL=http://localhost:11434

# Токен телеграм бота
TELEGRAM_BOT_TOKEN=123

# Параметры LLM
REASONING=False
CONTEXT_LENGTH=8192
CHAT_MESSAGES_LIMIT=10
TEMPERATURE=0.2
SEED=42
NUM_PREDICT=2048

# Параметры чанкера
CHUNK_SIZE=1500
CHUNK_OVERLAP=200

# Параметры поиска
FAISS_K=10
BM25_K=10
FAISS_WEIGHT=0.6
BM25_WEIGHT=0.4
```

---

| Параметр              | Значение по умолчанию      | Описание                                                                                                             | Описание значения                                                                                                  |
|-----------------------|----------------------------|----------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------|
| `EMBEDDING_MODEL`     | `nomic-embed-text`         | Модель для создания векторных представлений (эмбеддингов) текста. Используется для поиска и ранжирования.            | Строка — имя модели, поддерживаемой Ollama (например, `nomic-embed-text`, `all-minilm:l6-v2`, `mxbai-embed-large`) |
| `LLM_MODEL`           | `FieldMouse-AI/qwen3.5:9b` | Основная языковая модель (LLM) для генерации ответов. Указывается имя модели в Ollama.                               | Строка — имя LLM-модели в Ollama (например, `qwen2.5:1.5b`, `llama3.2:3b`, `mistral:7b`)                           |
| `OLLAMA_BASE_URL`     | `http://localhost:11434`   | URL-адрес сервера Ollama, к которому подключается система для выполнения запросов к моделям.                         | Строка — корректный URL (например, `http://localhost:11434`, `http://192.168.1.2:11434`)                           |
| `TELEGRAM_BOT_TOKEN`  | `123`                      | Токен авторизации Telegram-бота. Используется для подключения к API Telegram и получения обновлений.                 | Строка — валидный токен бота Telegram                                                                              |
| `REASONING`           | `False`                    | Включает или отключает режим "рассуждения" (Chain-of-Thought) при генерации ответов (не каждая модель поддерживает). | Булево: `"True"` или `"False"`                                                                                     |
| `CONTEXT_LENGTH`      | `8192`                     | Максимальная длина контекста (в токенах), которая учитывается моделью при генерации.                                 | Целое положительное число                                                                                          |
| `CHAT_MESSAGES_LIMIT` | `10`                       | Максимальное количество последних сообщений в истории диалога, передаваемых в модель.                                | Целое положительное число                                                                                          |
| `TEMPERATURE`         | `0.2`                      | Степень случайности генерации. Чем ниже значение, тем более детерминированные и консервативные ответы.               | Число с плавающей точкой в диапазоне `0.0-1.0`                                                                     |
| `SEED`                | `42`                       | Зерно для инициализации генератора случайных чисел, обеспечивает воспроизводимость результатов.                      | Любое целое число                                                                                                  |
| `NUM_PREDICT`         | `2048`                     | Максимальное количество токенов, которые модель может сгенерировать в одном ответе.                                  | Целое положительное число                                                                                          |
| `CHUNK_SIZE`          | `1500`                     | Размер одного фрагмента текста (в символах) при разбиении документов перед индексацией.                              | Целое положительное число                                                                                          |
| `CHUNK_OVERLAP`       | `200`                      | Количество символов перекрытия между соседними чанками, чтобы сохранить контекстную целостность.                     | Целое положительное число, меньшее `CHUNK_SIZE`                                                                    |
| `FAISS_K`             | `10`                       | Количество наиболее релевантных фрагментов, извлекаемых с помощью FAISS (векторный поиск).                           | Целое положительное число                                                                                          |
| `BM25_K`              | `10`                       | Количество наиболее релевантных фрагментов, извлекаемых с помощью BM25 (лексический поиск).                          | Целое положительное число                                                                                          |
| `FAISS_WEIGHT`        | `0.6`                      | Вес (коэффициент) результатов FAISS при итоговом ранжировании (в сумме с BM25_WEIGHT должно давать 1.0).             | Число с плавающей точкой в диапазоне `0.0-1.0`                                                                     |
| `BM25_WEIGHT`         | `0.4`                      | Вес (коэффициент) результатов BM25 при итоговом ранжировании.                                                        | Число с плавающей точкой в диапазоне `0.0-1.0`                                                                     |
